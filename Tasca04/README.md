# 🔐 SSH (Secure Shell)

## 💡 Què és?

SSH és un protocol que permet connectar-se a sistemes remots de manera **segura i encriptada**.

S’utilitza principalment per:

* Administració de servidors Linux 🐧
* Execució de comandes remotes
* Transferència segura de fitxers
* Gestió de sistemes sense entorn gràfic

---

## ⚙️ Funcionament bàsic

```bash id="ssh1"
ssh usuari@ip_del_servidor
```

---

## 🔒 Avantatges

* Connexió xifrada 🔐
* Alta seguretat
* Lleuger i ràpid
* Ideal per servidors

---

## ⚠️ Inconvenients

* Requereix coneixements de terminal
* No té interfície gràfica

---

# 🖥️ RDP (Remote Desktop Protocol)

## 💡 Què és?

RDP és un protocol que permet controlar un ordinador remot amb **interfície gràfica completa**.

S’utilitza habitualment en:

* Servidors Windows
* Equips d’usuari amb entorn gràfic
* Suport tècnic remot

---

## ⚙️ Característiques

* Control total de l’escriptori remot
* Ús de ratolí i teclat com si fossis local
* Compatible amb Windows i clients Linux

---

# 🔐 Seguretat en l’accés remot

## 🛡️ Bones pràctiques

* Utilitzar contrasenyes fortes
* Activar autenticació per clau SSH
* Limitar accessos per IP
* Utilitzar VPN quan sigui possible
* Evitar ports per defecte
* Mantenir sistemes actualitzats

---

# 🌍 Importància professional

L’accés remot és essencial perquè permet:

* ⚡ Resolució ràpida d’incidències
* 🌐 Gestió de servidors a distància
* 💼 Treball en entorns cloud
* 🧑‍💻 Suport tècnic immediat a clients

Permet administrar sistemes de manera eficient sense necessitat d’estar físicament al lloc, millorant la velocitat de resposta i la qualitat del servei.

SSH i RDP són eines fonamentals que tot tècnic ha de dominar.
